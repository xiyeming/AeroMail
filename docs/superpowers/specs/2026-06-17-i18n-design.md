# AeroMail 国际化（i18n）设计规范

> **范围**：支持 English / 简体中文 两种界面语言；前后端用户可见文案统一由前端本地化。
> **目标**：语言偏好持久化、启动自动恢复、多处可切换、后端错误码由前端映射为本地化文本。

---

## 1. 总体架构

```text
┌─────────────────────────────────────────────────────────────┐
│ 前端 Vue 3                                                   │
│  - vue-i18n（Composition API）                               │
│  - useLocale composable（读取/切换/持久化）                   │
│  - 翻译文件 src/i18n/locales/{en,zh-CN}.json                │
│  - 错误码 → 翻译 key 映射                                     │
└─────────────────────────────────────────────────────────────┘
                              │
                              │ Tauri IPC
                              │
┌─────────────────────────────────────────────────────────────┐
│ 后端 Rust                                                    │
│  - settings 表保存 app.locale                                │
│  - AeroError 携带 ErrorCode + 参数                            │
│  - Tauri Command 返回 Result<T, ErrorPayload>               │
└─────────────────────────────────────────────────────────────┘
```

---

## 2. 设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 前端框架 | `vue-i18n` v10 | Vue 3 官方生态，支持 Composition API 和懒加载 |
| 后端本地化方式 | 返回错误码 + 参数 | 翻译集中在前端，避免前后端文案重复维护 |
| 语言偏好存储 | SQLite `settings` 表 `app.locale` | 与现有设置体系一致，持久化 |
| 默认语言策略 | 先读设置，无设置则检测 OS 语言，最终回退 en | 首次使用体验友好 |
| 切换入口 | Settings / Status Bar / Command Palette | 满足不同场景快速切换 |
| 翻译文件格式 | JSON | 无需额外解析器，与 vue-i18n 默认格式一致 |

```typescript
export async function invokeLocalized<T>(cmd: string, args?: unknown): Promise<T> {
  const { t } = useI18n();
  try {
    return await invoke<T>(cmd, args);

    throw new Error(t(`errors.${payload.code}`, payload.args));
```

## 5. 前端实现

### 5.1 i18n 入口

```typescript
// src/i18n/index.ts
import { createI18n } from 'vue-i18n';

export type Locale = 'en' | 'zh-CN';

export const SUPPORTED_LOCALES: Locale[] = ['en', 'zh-CN'];

export const i18n = createI18n({
  legacy: false,
  locale: 'en',
  fallbackLocale: 'en',
  messages: {},
});

export async function loadLocaleMessages(locale: Locale): Promise<void> {
  const messages = await import(`./locales/${locale}.json`);
  i18n.global.setLocaleMessage(locale, messages.default);
}
```

### 5.2 locale composable

```typescript
// src/composables/useLocale.ts
import { ref, watch } from 'vue';
import { useI18n } from 'vue-i18n';
import type { Locale } from '@/i18n';
import { SUPPORTED_LOCALES, loadLocaleMessages } from '@/i18n';
import { useSettingsStore } from '@/stores/settings';

export function useLocale() {
  const { locale } = useI18n();
  const settings = useSettingsStore();
  const isReady = ref(false);

  async function initLocale() {
    const saved = await settings.get('app.locale');
    const target = resolveLocale(saved);
    await setLocale(target, false);
    isReady.value = true;
  }

  async function setLocale(value: Locale, persist = true) {
    if (!SUPPORTED_LOCALES.includes(value)) return;
    await loadLocaleMessages(value);
    locale.value = value;
    document.documentElement.setAttribute('lang', value);
    if (persist) {
      await settings.set('app.locale', value);
    }
  }

  function resolveLocale(raw: string | null): Locale {
    if (raw && SUPPORTED_LOCALES.includes(raw as Locale)) return raw as Locale;
    const osLang = navigator.language;
    if (osLang.startsWith('zh')) return 'zh-CN';
    return 'en';
  }

  return {
    locale,
    isReady,
    initLocale,
    setLocale,
    supportedLocales: SUPPORTED_LOCALES,
  };
}
```

### 5.3 Settings Store

```typescript
// src/stores/settings.ts
import { defineStore } from 'pinia';
import { invoke } from '@tauri-apps/api/core';

export const useSettingsStore = defineStore('settings', () => {
  async function set(key: string, value: string): Promise<void> {
    await invoke('set_setting', { key, value });
  }

  async function get(key: string): Promise<string | null> {
    return await invoke('get_setting', { key });
  }

  return { set, get };
});
```

### 5.4 LocaleSwitch 组件


## 6. 后端实现

### 6.1 改造 AeroError

```rust
// src-tauri/src/error.rs
use crate::models::error::ErrorPayload;

#[derive(Debug)]
pub enum AeroError {
    ImapAuthFailed { account: String },
    ImapConnectionTimeout,
    SmtpSendFailed { reason: String },
    InvalidAccountConfig { field: String },
    DatabaseError(rusqlite::Error),
    EncryptionError,
    Other(String),
}

impl AeroError {
    pub fn to_payload(&self) -> ErrorPayload {
        match self {
            AeroError::ImapAuthFailed { account } => ErrorPayload {
                code: "IMAP_AUTH_FAILED".to_string(),
                args: vec![account.clone()],
            },
            AeroError::ImapConnectionTimeout => ErrorPayload {
                code: "IMAP_CONNECTION_TIMEOUT".to_string(),
                args: vec![],
            },
            AeroError::SmtpSendFailed { reason } => ErrorPayload {
                code: "SMTP_SEND_FAILED".to_string(),
                args: vec![reason.clone()],
            },
            AeroError::InvalidAccountConfig { field } => ErrorPayload {
                code: "INVALID_ACCOUNT_CONFIG".to_string(),
                args: vec![field.clone()],
            },
            AeroError::DatabaseError(_) => ErrorPayload {
                code: "DATABASE_ERROR".to_string(),
                args: vec![],
            },
            AeroError::EncryptionError => ErrorPayload {
                code: "ENCRYPTION_ERROR".to_string(),
                args: vec![],
            },
            AeroError::Other(msg) => ErrorPayload {
                code: "UNKNOWN_ERROR".to_string(),
                args: vec![msg.clone()],
            },
        }
    }
}
```

### 6.2 Settings 表操作

复用现有 `settings` 表结构（key/value）。新增 `set_setting` / `get_setting` 命令：

```rust
// src-tauri/src/commands/settings.rs
use crate::state::AppState;
use crate::models::error::ErrorPayload;

#[tauri::command]
pub fn set_setting(
    key: String,
    value: String,
    state: tauri::State<'_, AppState>,
) -> Result<(), ErrorPayload> {
    state.db.set_setting(&key, &value).map_err(|e| e.to_payload())
}

#[tauri::command]
pub fn get_setting(
    key: String,
    state: tauri::State<'_, AppState>,
) -> Result<Option<String>, ErrorPayload> {
    state.db.get_setting(&key).map_err(|e| e.to_payload())
}
```

## 8. 安全与性能

- **XSS**：翻译文件只包含纯文本，不解析 HTML；如需富文本插值，使用 vue-i18n 的 `<i18n-t>` 组件。
- **性能**：locale JSON 按语言懒加载，不增加首屏体积。
- **持久化**：settings 表已存在，无需新增表。
